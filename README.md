## 🖥️ What this app is

Agent Reliability Engineering helps you work with AI agent systems in a safer, more controlled way. It focuses on checks, results, and tracking so you can see how your agent behaves over time.

Use it to:
- test agent runs
- track quality changes
- compare versions
- review performance metrics
- run transfer tests across setups
- keep config changes organized

## 📌 Main features

### 🧪 Evals
Run repeatable tests on agent behavior. This helps you check if changes improve or hurt results.

### 📊 imp@k metrics
Review output quality with ranked metrics that focus on the first few results. This helps when only the top answers matter.

### 🔁 Self-improvement tracking
Watch how the agent changes after updates. This makes it easier to see if a change helped.

### 🗂️ Config versioning
Keep versions of your setup files. You can compare old and new settings without losing track.

### 🔄 Transfer experiments
Test how one agent setup performs in a different task or environment.

### 👥 Multi-agent support
Work with more than one agent in the same flow. This helps when tasks need separate roles.

## 🔍 What to look at in results

When you review a run, focus on:
- task success rate
- top-ranked answer quality
- error patterns
- time per run
- config changes that affect output
- differences between agent versions

These checks help you learn which setup works best.

## 🛠️ Troubleshooting

### The file does not open
- Make sure the download finished
- Check that you extracted the zip file
- Try right-clicking the file and choosing **Run as administrator**

### Windows blocks the app
- Choose **More info** if it appears
- Then choose **Run anyway**
- Check that you downloaded it from the GitHub page above

### The app closes right away
- Open the app again
- Look for a log file in the `logs` folder
- Check that any config file you selected is in the correct place

### The app shows missing files
- Keep the full folder together
- Do not move one file without the others
- Extract the whole archive again if needed

### The screen looks empty
- Wait a moment for the first load
- Refresh or reopen the app
- Check whether sample data is needed for the first run
